# Migración de Datos a S3

Este documento describe cómo SensorWave maneja la migración de datos entre los nodos edge y el almacenamiento S3 en la nube.

## Configuración S3

SensorWave soporta cualquier almacenamiento S3-compatible:
- AWS S3
- Cloudflare R2
- MinIO
- Garage
- Otros compatibles con S3 API

### Parámetros de configuración

```go
tipos.ConfiguracionS3{
    Endpoint:        "https://...",  // URL del servidor S3
    AccessKeyID:     "...",          // Access Key ID
    SecretAccessKey: "...",          // Secret Access Key
    Bucket:          "sensorwave",   // Nombre del bucket
    Region:          "auto",         // Región (default: "us-east-1")
}
```

## Modos de operación

### Modo con nube (ConfigS3 != nil)

Cuando se proporciona configuración S3 al crear el ManagerEdge:

1. Se configura la conexión S3
2. Se verifica/crea el bucket automáticamente
3. Se registra el nodo en S3
4. Cada nueva serie se registra en S3

### Modo local (ConfigS3 == nil)

Cuando no se proporciona configuración S3:

1. El nodo funciona completamente offline
2. Los datos permanecen en PebbleDB local
3. No hay registro del nodo en la nube
4. La función `MigrarAS3()` fallará si no hay credenciales en variables de entorno

## Estructura de datos en S3

### Registro de nodos

**Ubicación**: `nodos/<nodoID>.json`

```json
{
    "nodo_id": "edge-hostname-uuid",
    "direccion_ip": "192.168.1.100",
    "puerto_http": "8080",
    "series": {
        "invernadero/temperatura": {
            "path": "invernadero/temperatura",
            "tipo_datos": "Real",
            "compresion_bloque": "ZSTD",
            "compresion_bytes": "Xor",
            "tamaño_bloque": 20,
            "serie_id": 1,
            "tags": {"unidad": "celsius"}
        }
    }
}
```

El registro se actualiza cuando:
- Se crea el nodo por primera vez
- Se agrega una nueva serie

### Datos de series

**Ubicación**: `<nodoID>/data/<serieID>/<timestamp_inicio>_<timestamp_fin>`

Ejemplo:
```
edge-cbiale-abc123/data/0000000001/01765707284149915667_01765707303149915667
```

Donde:
- `edge-cbiale-abc123`: ID único del nodo
- `0000000001`: ID de la serie (10 dígitos, padding con ceros)
- `01765707284149915667`: Timestamp de inicio en nanosegundos (20 dígitos)
- `01765707303149915667`: Timestamp de fin en nanosegundos (20 dígitos)

Los datos se almacenan comprimidos en formato binario (igual que en PebbleDB local).

## Flujo de creación de series

```
CrearSerie(config)
    │
    ├─► Validar configuración
    │
    ├─► Guardar en PebbleDB local
    │       └─► series/<path>
    │
    ├─► Crear buffer en memoria
    │
    └─► Si clienteS3 != nil
            │
            └─► RegistrarEnS3()
                    │
                    ├─► Serializar nodo + series a JSON
                    │
                    └─► PUT nodos/<nodoID>.json
                            │
                            ├─► Éxito: log informativo
                            │
                            └─► Error: log de advertencia
                                (la serie SÍ se crea localmente)
```

## Flujo de migración

```
MigrarAS3()
    │
    ├─► ¿clienteS3 configurado?
    │       │
    │       ├─► No: Intentar configurar desde variables de entorno
    │       │       │
    │       │       ├─► S3_ENDPOINT
    │       │       ├─► S3_ACCESS_KEY_ID
    │       │       ├─► S3_SECRET_ACCESS_KEY
    │       │       ├─► S3_BUCKET
    │       │       └─► S3_REGION
    │       │
    │       └─► Si faltan credenciales: ERROR (retorna)
    │
    ├─► Iterar sobre datos en PebbleDB
    │       └─► Prefijo: data/
    │
    └─► Por cada bloque de datos:
            │
            ├─► PUT <nodoID>/<clave> en S3
            │       │
            │       ├─► Éxito: DELETE de PebbleDB local
            │       │
            │       └─► Error: DETENER migración (retorna error)
            │
            └─► Log cada 100 registros
```

## Tolerancia a fallos

### Al crear el nodo (Crear)

| Escenario | Comportamiento | Resultado |
|-----------|----------------|-----------|
| S3 no configurado | Modo local activado | Nodo funciona offline |
| Error de conexión S3 | Log de advertencia | Nodo funciona en modo local |
| Error al registrar nodo | Log de advertencia | Nodo funciona en modo local |
| Bucket no existe | Se intenta crear automáticamente | Bucket creado o error |

**Código relevante** (`edge.go:163-184`):
```go
if opts.ConfigS3 != nil {
    err = manager.ConfigurarS3(*opts.ConfigS3)
    if err != nil {
        log.Printf("Advertencia: error al configurar S3: %v", err)
        log.Printf("El nodo funcionará en modo local")
    } else {
        err = manager.RegistrarEnS3()
        if err != nil {
            log.Printf("Advertencia: error registrando nodo en S3: %v", err)
        }
    }
} else {
    log.Printf("Modo local: sin registro en nube (S3 no configurado)")
}
```

### Al crear series (CrearSerie)

| Escenario | Comportamiento | Resultado |
|-----------|----------------|-----------|
| S3 no configurado | No se intenta registrar | Serie creada localmente |
| Error al registrar en S3 | Log de advertencia | Serie creada localmente |
| S3 configurado, conexión OK | Registro actualizado | Serie disponible local y nube |

**Código relevante** (`edge.go:394-400`):
```go
if clienteS3 != nil {
    err = me.RegistrarEnS3()
    if err != nil {
        log.Printf("Error registrando serie nueva en S3: %v", err)
    }
}
```

**Importante**: La serie SIEMPRE se crea localmente, independientemente del estado de S3.

### Al migrar datos (MigrarAS3)

| Escenario | Comportamiento | Resultado |
|-----------|----------------|-----------|
| S3 no configurado | Intenta configurar desde env vars | Error si faltan credenciales |
| Error de conexión | Migración se detiene | Error retornado, datos locales intactos |
| Error al subir objeto | Migración se detiene | Error retornado, datos locales intactos |
| Error al borrar local | Migración se detiene | Dato puede quedar duplicado |

**Código relevante** (`migracion_datos.go:147-161`):
```go
_, err := clienteS3.PutObject(ctx, &s3.PutObjectInput{...})
if err != nil {
    return fmt.Errorf("error al subir dato a S3 (clave: %s): %v", clave, err)
}

contadorMigrados++

err = me.db.Delete(iter.Key(), pebble.Sync)
if err != nil {
    return fmt.Errorf("error al borrar dato migrado de PebbleDB: %v", err)
}
```

**Importante**: 
- Si falla la subida, el dato NO se borra localmente
- Si falla el borrado local, el dato queda duplicado (en S3 y local)
- La migración es atómica por objeto, no por lote

## Resumen de tolerancia a fallos

| Operación | Falla S3 | Comportamiento |
|-----------|----------|----------------|
| `Crear()` con ConfigS3 | Sí | Nodo funciona en modo local |
| `Crear()` sin ConfigS3 | N/A | Modo local desde el inicio |
| `CrearSerie()` | Sí | Serie creada localmente, no registrada en S3 |
| `MigrarAS3()` | Sí | Error retornado, datos locales preservados |
| `RegistrarEnS3()` | Sí | Error retornado, nodo funciona normalmente |

## Variables de entorno

Si no se proporciona ConfigS3 pero se llama a `MigrarAS3()`:

| Variable | Descripción | Default |
|----------|-------------|---------|
| `S3_ENDPOINT` | URL del servidor S3 | `http://localhost:3900` |
| `S3_ACCESS_KEY_ID` | Access Key ID | (requerido) |
| `S3_SECRET_ACCESS_KEY` | Secret Access Key | (requerido) |
| `S3_BUCKET` | Nombre del bucket | `sensorwave-data` |
| `S3_REGION` | Región S3 | `us-east-1` |

## Ejemplo de uso

```go
// Con configuración S3 explícita
configS3 := tipos.ConfiguracionS3{
    Endpoint:        "https://xxx.r2.cloudflarestorage.com",
    AccessKeyID:     "mi-access-key",
    SecretAccessKey: "mi-secret-key",
    Bucket:          "sensorwave",
    Region:          "auto",
}

manager, err := edge.Crear(tipos.OpcionesEdge{
    NombreDB:   "mi_base.db",
    PuertoHTTP: "8080",
    ConfigS3:   &configS3,
})

// Crear series (se registran automáticamente en S3)
manager.CrearSerie(tipos.Serie{
    Path:             "sensor/temperatura",
    TipoDatos:        tipos.Real,
    CompresionBloque: tipos.ZSTD,
    CompresionBytes:  tipos.Xor,
    TamañoBloque:     100,
})

// Insertar datos (se almacenan localmente)
manager.Insertar("sensor/temperatura", time.Now().UnixNano(), 25.5)

// Migrar datos a S3 cuando sea necesario
err = manager.MigrarAS3()
if err != nil {
    log.Printf("Error en migración: %v", err)
}
```
